## 🎯 核心特性

### AT模式特性
1. **自动补偿** - 无需手动编写回滚逻辑
2. **透明化** - 对业务代码侵入小
3. **前后镜像** - 自动生成数据快照
4. **Undo Log** - 自动管理回滚日志
5. **全局锁** - 保证隔离性

### TCC模式特性
1. **手动补偿** - 精确控制资源
2. **高性能** - 无镜像生成开销
3. **资源冻结** - 预留资源机制
4. **幂等保证** - 防止重复执行
5. **悬挂处理** - 完善的异常处理

## 🔄 工作流程

### AT模式流程
```
1. TM开启全局事务，生成XID
2. RM执行SQL前查询前镜像
3. RM执行业务SQL
4. RM查询后镜像，生成undo_log
5. RM提交本地事务（一阶段）
6. TM决定全局提交或回滚
7. RM删除undo_log或执行回滚SQL（二阶段）
```

### TCC模式流程
```
1. TM开启全局事务，生成XID
2. Try阶段：冻结资源（如冻结余额）
3. 业务逻辑执行
4. TM决定全局提交或回滚
5a. Confirm：正式扣减资源
5b. Cancel：解冻资源
```
