# Token Bucket（令牌桶）

## 概述

令牌桶是一种流量控制算法，用于限制请求的速率。它通过维护一个固定容量的桶来存储令牌，令牌以固定的速率生成。每个请求需要消耗一个或多个令牌，只有当桶中有足够的令牌时，请求才会被允许。

## 算法原理

1. **初始化**：创建一个固定容量的桶，初始时桶是满的
2. **令牌生成**：以固定速率向桶中添加令牌，如果桶满了，新生成的令牌会被丢弃
3. **请求处理**：
   - 当请求到来时，尝试从桶中获取令牌
   - 如果桶中有足够的令牌，则移除相应数量的令牌，请求通过
   - 如果桶中令牌不足，请求被拒绝

## 特点

### 优点
- **支持突发流量**：桶中可以累积令牌，允许短时间内的突发请求
- **平滑限流**：长期来看，请求速率稳定在令牌生成速率
- **灵活性高**：可以动态调整容量和生成速率

### 缺点
- **内存开销**：需要维护令牌桶状态
- **时间精度**：依赖系统时间，可能受时钟调整影响

## 使用方法

### 基本用法

```go
package main

import (
	"fmt"
	"time"
	"awesomeProject/token_bucket"
)

func main() {
	// 创建容量为 10，每秒生成 5 个令牌的令牌桶
	tb := token_bucket.NewTokenBucket(10, 5)
	
	// 尝试获取一个令牌
	if tb.Allow() {
		fmt.Println("请求通过")
	} else {
		fmt.Println("请求被限流")
	}
	
	// 查看当前可用令牌数
	fmt.Printf("可用令牌数: %d\n", tb.AvailableTokens())
}
```

### 批量获取令牌

```go
// 一次性获取 5 个令牌
if tb.AllowN(5) {
	fmt.Println("批量请求通过")
} else {
	fmt.Println("令牌不足，请求被拒绝")
}
```

### 模拟流量控制

```go
tb := token_bucket.NewTokenBucket(10, 2) // 每秒 2 个令牌

for i := 0; i < 20; i++ {
	if tb.Allow() {
		fmt.Printf("请求 %d: 通过\n", i+1)
	} else {
		fmt.Printf("请求 %d: 被限流\n", i+1)
	}
	time.Sleep(100 * time.Millisecond)
}
```

## API 文档

### NewTokenBucket(capacity, refillRate int64) *TokenBucket

创建一个新的令牌桶。

**参数：**
- `capacity`：桶的容量（最大令牌数）
- `refillRate`：令牌生成速率（令牌数/秒）

**返回：**
- 令牌桶实例

### Allow() bool

尝试获取一个令牌。

**返回：**
- `true`：获取成功
- `false`：令牌不足，获取失败

### AllowN(n int64) bool

尝试获取 n 个令牌。

**参数：**
- `n`：需要获取的令牌数

**返回：**
- `true`：获取成功
- `false`：令牌不足，获取失败

### AvailableTokens() int64

返回当前可用的令牌数。

**返回：**
- 当前可用令牌数

### Capacity() int64

返回桶的容量。

**返回：**
- 桶的容量

### RefillRate() int64

返回令牌生成速率。

**返回：**
- 令牌生成速率（令牌数/秒）

## 应用场景

1. **API 限流**：限制 API 接口的调用频率
2. **流量整形**：控制网络流量的发送速率
3. **资源保护**：防止系统过载
4. **公平调度**：在多租户系统中实现公平的资源分配

## 与漏桶算法的区别

| 特性 | 令牌桶 | 漏桶 |
|------|--------|------|
| 突发流量 | 支持 | 不支持 |
| 流量整形 | 较弱 | 较强 |
| 实现复杂度 | 中等 | 简单 |
| 适用场景 | API 限流 | 流量整形 |

## 线程安全

本实现使用 `sync.Mutex` 保证并发安全，可以在多个 goroutine 中安全使用。
