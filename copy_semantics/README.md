# Go 的 `copy`：深拷贝还是浅拷贝？

## 结论（先说答案）

- `copy` 是**逐元素复制**。
- 对“纯值元素”切片，看起来像深拷贝（修改源不影响目标）。
- 对“引用语义元素”（如 `[]T` 里的 `T` 包含指针/slice/map 等），只复制引用，底层对象共享，表现为浅拷贝。

所以它不是绝对深拷贝，也不是绝对浅拷贝，取决于元素类型。

## 案例

- `[]int`：`copy` 后互不影响（值语义）
- `[][]int`：`copy` 后内层共享（引用语义）
- `[]struct{ Ptr *Node }`：指针目标共享（浅层）

## 何时需要“真深拷贝”？

当元素含引用字段且你需要完全隔离时，必须手写深拷贝逻辑（本项目提供 `DeepCopy2DInt` 示例）。
