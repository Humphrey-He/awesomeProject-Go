# Reflect 内部原理与实战用法

## Reflect 核心概念

Go 的 `reflect` 包提供了运行时类型检查和动态操作能力，是很多框架的基础：

- **序列化/反序列化**：JSON、XML 序列化
- **ORM**：数据库映射
- **依赖注入**：DI 容器
- **配置绑定**：Viper 等配置库

## 本模块内容

### 一、内部原理

- **rtype 结构**：Go 运行时的类型信息表示
- **Value 结构**：值的运行时包装
- **interface{} 内存布局**：类型指针 + 数据指针
- **指针操作**：Elem、Set、Int 等方法

### 二、实战用法

- 类型检查与转换
- 动态访问字段
- 动态调用方法
- 动态创建对象
- 深拷贝
- 标签解析
- 序列化/反序列化
- 零值与空值判断

## 核心函数

- `reflect.TypeOf(x)` - 获取类型信息
- `reflect.ValueOf(x)` - 获取值信息
- `v.Elem()` - 获取指针/接口的实际值
- `v.FieldByName()` - 获取字段
- `v.MethodByName()` - 获取方法
- `v.Call()` - 调用方法

## 生产实践建议

- **避免频繁反射**：反射有性能开销，热路径避免使用
- **缓存 Type 信息**：使用 map 缓存类型信息
- **检查 CanSet**：修改值前检查可设置性
- **优先具体类型**：能用具体类型就不用反射
